# 远山：能受用一生的 Obsidian 完整文件规划

> 一套以「二字纲目 + 古典短题 · 现代释义」为核心的长期知识库目录方案。

「远山」不是一个为了显得复杂而复杂的 Obsidian 模板，也不是把所有资料简单分门别类的文件夹清单。

它更像一座长期心智系统：安顿心志，整理经验，保存判断，锻造方法，并将所思所学逐渐化为作品与行动。

远山之中：

- 立心以定其本
- 修身以厚其力
- 经世以入其用
- 器用以广其能
- 著述以成其形
- 法度以立其常
- 采摭以纳外物
- 归藏以存旧事

这套目录适合希望长期使用 Obsidian 的人，尤其适合把个人成长、事业判断、AI 工具、项目经验、写作输出和方法论沉淀放在同一个系统里的人。

## 一、它与常见 Obsidian 目录有什么不同

很多 Obsidian 目录会从资料类型出发：

```text
读书笔记/
项目资料/
日记/
文章/
课程/
素材/
归档/
```

这种结构容易上手，但长期使用后常会出现几个问题。

第一，资料被分类了，但人没有被组织起来。你知道一条笔记来自哪本书、哪个项目、哪个课程，却不一定知道它最终服务于你的什么判断、什么行动、什么作品。

第二，文件夹越来越像仓库。东西都放进去了，但很少再被重新调动。Obsidian 最有价值的地方不是保存资料，而是让资料反复回到你的问题、决策和表达里。

第三，命名过于工具化。比如「AI 工具」「人生感悟」「商业资料」「方法论」当然清楚，但它们很容易变成临时分类，缺少长期系统感，也缺少一种愿意反复打开的审美吸引力。

第四，很多目录只解决“放在哪里”，不解决“为什么记、如何用、何时转化”。真正受用一生的知识库，应该同时回答三个问题：

- 这条内容属于什么主题？
- 它服务于我的什么成长、判断或行动？
- 它未来能否转化为方法、作品、项目或决策？

「远山」的不同之处在于，它不是按资料来源组织，而是按人的长期生命结构组织。

它把个人知识库拆成八个长期维度：

| 维度 | 解决的问题 |
|---|---|
| 立心 | 我是谁，我为何而行 |
| 修身 | 我如何增长自身 |
| 经世 | 我如何进入现实 |
| 器用 | 我如何驾驭工具 |
| 著述 | 我如何形成作品 |
| 法度 | 我如何沉淀方法 |
| 采摭 | 我如何吸纳外部 |
| 归藏 | 我如何安放旧事 |

这让 Obsidian 不只是一个资料库，而成为一个长期可更新的自我操作系统。

## 二、命名系统：二字纲目 + 古典短题 · 现代释义

「远山」采用一套克制、清晰、可久看的命名方式：

```text
顶层文件夹：二字纲目
具体笔记：古典短题 · 现代释义
```

例如：

```text
01_立心/
├── 立心 · 精神内核.md
├── 明志 · 长期志向.md
├── 自胜 · 自我要求.md
└── 知止 · 边界与取舍.md
```

前半部分「立心」「明志」「自胜」「知止」提供气韵、秩序和记忆点。

后半部分「精神内核」「长期志向」「自我要求」「边界与取舍」保证清晰、可搜索、可理解，也方便 AI 工具识别笔记用途。

### 这套命名的好处

**1. 它比普通分类更有长期感**

「人生感悟」容易变成情绪堆积，「立心」则提醒你写的是根本信念、价值排序和长期方向。

「AI 工具」容易变成软件清单，「器用」则提醒你工具要服务于判断、表达、执行和创造。

「方法论」容易泛化，「法度」则提醒你写的是可重复、可执行、可检验的规则。

**2. 它比口号式命名更克制**

远山避免使用「破局」「逆袭」「乘风破浪」「一站成名」这类强刺激词。

这些词第一次看可能很有力量，但三年后再看，往往容易显得焦躁、用力或过时。

远山追求的是含蓄、稳重、可久看。它不靠标题制造热血，而靠结构承载长期生活。

**3. 它兼顾审美与实用**

纯古典命名容易看不懂，纯现代命名又容易太平。

「古典短题 · 现代释义」的好处是：

- 古典短题负责气质与记忆
- 现代释义负责清晰与检索
- 中间的 `·` 让标题稳定、干净、不过度网络化

例如：

```text
假物 · AI时代的个人杠杆
机杼 · Codex能力沉淀
驭机 · Agent工作准则
澄汰 · 信息筛选
化裁 · 从想法到作品
```

这些标题既不玄，也不俗。打开时有一点书卷气，使用时又不妨碍实际工作。

**4. 它天然适合与 AI 协作**

现代释义部分能让 AI 明白笔记用途。

当你让 AI 帮你整理、扩写、生成索引、建立关联时，`立心 · 精神内核`、`经略 · 商业判断`、`程式 · SOP与标准化` 这类标题比纯粹的古典词更容易被理解。

这是一套适合人读、也适合 AI 协作的命名系统。

## 四、各目录说明

### 00_总目

总览、索引、年度主线、灵感暂存。

它是远山的目录与山门，不承担过多内容，只负责让整个系统有入口、有方向、有主线。

| 笔记 | 用途 |
|---|---|
| 总叙 · 远山总览 | 整个 Vault 首页 |
| 纲目 · 知识地图 | 所有主题地图的汇总 |
| 岁序 · 年度主线 | 每年的主线、阶段与方向 |
| 一得 · 灵感暂存 | 临时灵感、片段想法 |
| 旧札 · Apple备忘录精选 | 旧感悟与旧笔记的迁移入口 |

### 01_立心

精神内核、志向、原则、定力。

这里不写一时情绪，而写你的根本信念、人生底色和价值排序。

| 笔记 | 用途 |
|---|---|
| 立心 · 精神内核 | 核心信念、价值观、人生底色 |
| 明志 · 长期志向 | 想成为什么样的人，想创造什么 |
| 自胜 · 自我要求 | 高标准、内在驱动力 |
| 进德 · 向上生长 | 持续成长与精进 |
| 知止 · 边界与取舍 | 什么值得做，什么不值得做 |
| 持正 · 原则与底线 | 不能妥协的原则 |
| 临事 · 直面现实 | 面对真实问题的姿态 |
| 处变 · 周期中的定力 | 逆境、波动、变化中的稳定性 |

### 02_修身

学习、精力、习惯、判断力、自我训练。

它不是“我要变好”的口号区，而是把身体、精力、学习、注意力和判断力沉淀为系统。

| 笔记 | 用途 |
|---|---|
| 日新 · 自我进化 | 自我成长总入口 |
| 厚积 · 能力复利 | 技能、经验、长期积累 |
| 慎独 · 自我统御 | 注意力、纪律、欲望管理 |
| 静定 · 注意力与心力 | 深度工作、专注、内在秩序 |
| 养气 · 身体与精力 | 睡眠、运动、精力、体能 |
| 藏修 · 学习系统 | 课程、学习计划、长期训练 |
| 博观 · 阅读与外部世界 | 阅读、英文信息、海外视野 |
| 慎思 · 判断力训练 | 复杂问题分析、观点形成 |

### 03_经世

事业、商业、项目、团队、趋势。

这里不只写工作任务，而写你如何进入现实世界，经营判断、资源、项目和价值。

| 笔记 | 用途 |
|---|---|
| 经世 · 事业总览 | 事业系统总入口 |
| 观势 · 行业趋势 | AI、跨境电商、SaaS、市场变化 |
| 经略 · 商业判断 | 商业模式、客户、市场、变现逻辑 |
| 度势 · 项目决策 | 项目是否做、怎么做、何时停 |
| 取舍 · 机会筛选 | 哪些机会值得投入 |
| 识人 · 团队与协作 | 人才判断、授权、协作机制 |
| 成务 · 业务系统化 | 把经验转化成流程、产品、组织能力 |

### 04_器用

AI-Lab、Codex、Agent、NAS、工具系统。

工具不是玩具。工具服务于你的判断、表达、执行和创造。

| 笔记 | 用途 |
|---|---|
| 器用 · AI-Lab总览 | AI-Lab 总入口 |
| 假物 · AI时代的个人杠杆 | 为什么要使用 AI 与 Agent |
| 机杼 · Codex能力沉淀 | Codex 使用经验、代码、自动化 |
| 驭机 · Agent工作准则 | Agent 的角色、边界、流程、规范 |
| 设问 · Prompt范式 | Prompt 模板、提问结构、任务指令 |
| 枢机 · Mac mini工作站 | Mac mini 配置、远程工作站、运行环境 |
| 藏器 · NAS资产仓库 | NAS 目录、素材、归档、备份 |
| 通远 · Tailscale远程网络 | 远程访问、网络连接、节点说明 |
| 守密 · 安全与密钥原则 | 1Password、API Key、Token 管理原则 |

注意：安全类笔记只写规则，不写密钥、Token 或任何可直接使用的敏感信息。

### 05_著述

网站、文章、PPT、视频、观点表达。

知识库最终不能只停留在记笔记。它要帮助你形成作品、表达、方案、网站、视频和案例。

| 笔记 | 用途 |
|---|---|
| 著述 · 作品总览 | 所有输出成果的入口 |
| 门户 · 个人网站 | 个人网站结构、文案、更新记录 |
| 立言 · 文章与观点 | 长文、观点、网站内容、公众号 |
| 成篇 · PPT与方案 | PPT、汇报逻辑、方案结构 |
| 声影 · 短视频创作 | 视频脚本、画面、模型调用、流程 |
| 纪实 · 项目案例 | 成功案例、失败案例、项目沉淀 |
| 化裁 · 从想法到作品 | 灵感如何变成可发布成果 |

### 06_法度

方法论、复盘、决策、SOP、系统化。

这里的内容要可重复、可执行、可检验。不要把方法论写成鸡汤。

| 笔记 | 用途 |
|---|---|
| 法度 · 方法论总览 | 方法论总入口 |
| 知行 · 行动原则 | 从想法到行动的规则 |
| 反照 · 复盘方法 | 项目复盘、人生复盘、技术复盘 |
| 审机 · 决策框架 | 机会、风险、投入产出判断 |
| 纲纪 · 系统化思维 | 如何建立结构、流程、模型 |
| 程式 · SOP与标准化 | 可复用流程、执行标准 |
| 约取 · 知识内化 | 从输入中提炼真正有用的东西 |
| 澄汰 · 信息筛选 | 信息源管理、去噪、判断信息质量 |

### 07_采摭

书摘、视频笔记、文章摘录、案例研究。

外部资料不能直接等于你的观点。先采摭，再消化，再进入法度、经世或著述。

| 笔记 | 用途 |
|---|---|
| 采摭 · 资料总览 | 外部资料入口 |
| 观书 · 书籍摘录 | 读书笔记、书摘 |
| 观课 · 视频与课程笔记 | YouTube、课程、教程 |
| 摘录 · 文章与报告 | 网页、行业报告、长文摘要 |
| 取法 · 案例研究 | 优秀案例、典型经验 |
| 辞章 · 名句与标题素材 | 可引用诗文、典故、标题词库 |

### 99_归藏

已完成、暂停、过期资料。

归藏的作用是保持主系统清洁。完成不等于删除，暂停也不等于遗忘。

| 笔记 | 用途 |
|---|---|
| 归藏 · 已完成项目 | 完成项目的索引 |
| 待考 · 暂停事项 | 暂不推进但仍有价值的事项 |
| 旧稿 · 过期资料 | 旧方案、旧资料、旧笔记 |
| 存照 · 历史记录 | 重要变更、旧版本、迁移记录 |

## 六、如何让别人直接使用

### 方法一：手动创建

1. 在 Obsidian 中新建一个 Vault，命名为 `远山`。
2. 按照「完整目录结构」创建 9 个顶层文件夹。
3. 在每个文件夹中创建对应的 Markdown 笔记。
4. 将 `总叙 · 远山总览.md` 设为首页或加入书签。
5. 从 `一得 · 灵感暂存.md` 开始记录第一批想法。

适合第一次搭建的人，因为手动创建会帮助你理解每个目录的意义。

### 方法二：从最小版本开始

如果不想一次创建 62 篇笔记，可以先建下面 18 篇：

| 文件夹 | 第一批笔记 |
|---|---|
| 00_总目 | 总叙 · 远山总览、纲目 · 知识地图、一得 · 灵感暂存 |
| 01_立心 | 立心 · 精神内核、明志 · 长期志向、自胜 · 自我要求、进德 · 向上生长、临事 · 直面现实 |
| 02_修身 | 日新 · 自我进化、厚积 · 能力复利、慎独 · 自我统御 |
| 03_经世 | 经世 · 事业总览、经略 · 商业判断 |
| 04_器用 | 器用 · AI-Lab总览、假物 · AI时代的个人杠杆、机杼 · Codex能力沉淀 |
| 05_著述 | 著述 · 作品总览 |
| 06_法度 | 法度 · 方法论总览、知行 · 行动原则 |

最小版本的原则是：先立骨架，不急着搭空楼。

### 方法三：复制为自己的长期系统

使用者可以保留顶层结构，但替换部分笔记名称。

例如：

```text
器用 · AI-Lab总览
```

可以改成：

```text
器用 · 设计工具总览
器用 · 开发工具总览
器用 · 研究工具总览
```

但建议保留 `二字纲目 · 现代释义` 的命名方式。这样系统既有一致性，也有个人化空间。

## 七、建议的单篇笔记模板

每篇笔记可以使用下面的基础格式：

```markdown
---
status: seed
tags:
  - 远山/立心
---

# 立心 · 精神内核

> 你的核心信念、价值观、人生底色。

## 备注

这里写根本判断，不写一时情绪。

## 关联

- [[明志 · 长期志向]]
- [[自胜 · 自我要求]]
- [[知止 · 边界与取舍]]

## 可写内容

- 

## 待补问题

- 

## 相关素材

- 
```

`status: seed` 表示这是一篇种子笔记。未来可以改成：

```text
seed       初始种子
growing    持续生长
mature     已经成熟
archived   已归档
```

## 八、使用原则

### 1. 一念初起，先入暂存

任何突然出现的想法，不要急着找完美位置，先放进 `一得 · 灵感暂存.md`。

### 2. 反复出现，归入主题

一个想法如果反复出现，说明它不是噪音。把它迁入 `立心`、`修身`、`经世`、`器用`、`著述` 或 `法度` 中对应的主题笔记。

### 3. 可用于事，沉淀为法

如果一条经验能指导下一次行动，就不要只把它留在项目记录里，应提炼到 `法度` 中。

### 4. 可示于人，整理成篇

如果一条想法已经有表达价值，就转入 `著述`，整理成文章、PPT、视频脚本、网站内容或案例。

### 5. 已成旧事，归藏存照

完成、暂停、过期的内容，不必删除。放入 `99_归藏`，让主系统保持清洁。

## 九、这套系统适合谁

远山适合这些人：

- 想长期认真使用 Obsidian 的人
- 不满足于只收藏资料的人
- 希望把知识转化为判断、行动和作品的人
- 正在建设个人 AI 工作流、Agent 工作流或自动化系统的人
- 希望把事业、成长、写作、工具和方法论统一管理的人
- 喜欢中文命名、古典气韵和长期秩序感的人

它不适合这些场景：

- 只想临时存文件
- 只需要任务清单
- 不打算长期维护
- 只追求极简，不需要精神结构和审美系统

## 十、结语

「远山」的重点不在于文件夹多，也不在于名字雅。

它真正想解决的是：一个人如何在长期生活中保存自己的判断，训练自己的能力，面对现实事务，借助工具放大自身，把经验沉淀为方法，并最终形成作品。

一个好的 Obsidian 系统，不应该只是笔记的容器。

它应该是一种长期秩序。

愿远山可望，也可行。
